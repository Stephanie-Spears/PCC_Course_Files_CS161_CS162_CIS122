Overview
--------

The last chapter introduced the control structures used for selection.
This week we will look at the control structures used for
repetition. Just like the "if" statement used boolean expressions to
make decisions, the repetition (or looping) structures will use
expressions to control the loops.

There are three main looping constructs: while, for, do...while.

while Looping Structure
-----------------------

A while loop consists of an expression (or condition) and then the
statements that are controlled by the expression.  Consider this:

int i=0;                  // line 1
while (i<=20) {           // line 2
  cout << i << endl;      // line 3
  i = i + 5;              // line 4
}                         // line 5

The condition is "i<=20".  As long as that condition is true, the loop
body will run.  Let's look at this loop step by step.

In line 1 we set the variable i to 0.

In line 2 we evaluate the expression "i<=20".  Since i is 0 and 0 <
20, the expression is true.  So we will run one iteration of the
loop.  

Line 3 will print out "0".

Line 4 will add 5 to i so the new value of i is 5.

Line 5 is the end of the loop body so we go back to evaluate the
condition in line 2.  Since i is now 5 and is still <= 20 we do the
loop body again.

Line 3 will print out "5".

Line 4 will add 5 to i so the new value of i is 10.

Line 5 is the end of the loop body so we go back to evaluate the
condition in line 2.  Since i is now 10 and is still <= 20 we do the
loop body again.

Line 3 will print out "10".

Line 4 will add 5 to i so the new value of i is 15.

Line 5 is the end of the loop body so we go back to evaluate the
condition in line 2.  Since i is now 15 and is still <= 20 we do the
loop body again.

Line 3 will print out "15".

Line 4 will add 5 to i so the new value of i is 20.

Line 5 is the end of the loop body so we go back to evaluate the
condition in line 2.  Since i is now 20 and is still <= 20 we do the
loop body again.

Line 3 will print out "20".

Line 4 will add 5 to i so the new value of i is 25.

Line 5 is the end of the loop body so we go back to evaluate the
condition in line 2.  Since i is now 25, it is > 20 so we end the loop.

Types of while Loops
--------------------

It is useful to understand that there are different 'types' of while
loops.  From the C++ perspective they are all the same but the
difference is in how you think about it and how you use it.

The book describes these in detail.  Here is a summary:

- counter-controlled:  these loops have a condition that is based on a
  variable that increments or decrements.  The example above is an
  example of a counter-controlled loop.  With some analysis you can
  tell how many times the loop will run.

- sentinel-controlled: these loops use a "sentinel" value to tell the
  loop to exit.  For example, ask the user to enter data but enter a 0
  when there is no more data to enter.  You usually don't know how
  many times the loop will run until it is actually running.

- flag-controlled: these loops are similar to the sentinel loops but
  they use a boolean flag in the condition.  The flag is set in the
  loop body based on some event or condition.

There are other while loop types explained in the book but these are
the major ones.

for Looping Structure
---------------------

The while loop is general enough to do all repetitions but sometimes
it is convenient to have specific loop forms.  The "for" loop is a
form for doing a loop a specific number of times.

The examples in the book are very good at explaining these loops.  

In general, if you want a loop that runs a specific number of times,
use a for-loop.  If you want a loop that runs an unknown number of
times, use a while-loop.

do...while Looping Structure
----------------------------

The do...while loop is similar to the while loop but it checks the
condition AFTER the loop body runs.  The main affect of this is that
the loop body will always run at least once.  This is not the case
with while or for loops. In those loops it is possible that the
condition is false at the beginning of the loop so it won't run at
all.

Frequency of loops in code
--------------------------

In my experience of writing software I have found that the while loop
is the most common.  The second most common is the for loop.
do...while is seldom used.  Rough guesses put the relative frequencies
as follows:

while        65%
for        30%
do...while     5%

break and continue Statements
-----------------------------

The break and continue statements are way to alter the standard loop
execution. The break statement will immediately exit the loop (without
check the condition) and execution will continue after the loop body.

The continue statement will immediately stop the current iteration of
the loop and will jump to the condition.  This will be evaluated and
the loop will either continue or exit, depending on the condition.

Avoid using break and continue
------------------------------

Where possible it is best to avoid using the break and continue
statements.  These statement have their uses but usually they are
misused.  For best programming practice it is best to avoid them (when
possible).

Nested Control Structures
-------------------------

Just like with if statements, loop statements can be nested.  This
allows you to generate some fairly complex behavior.  Nested loops are
commonplace in real code so it is worth the effort to go through the
examples (5-18 through 5-23) and understand them.
