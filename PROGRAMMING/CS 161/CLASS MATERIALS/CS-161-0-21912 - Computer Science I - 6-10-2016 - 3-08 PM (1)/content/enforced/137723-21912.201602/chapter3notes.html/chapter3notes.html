Overview
--------

The focus of this chapter is on input and output.  We will focus
mostly on keyboard and script I/O but we will also look at file based
I/O.

I/O Streams and Standard I/O Devices
------------------------------------

This sections focuses on the extraction operator (>>).  Input in C++
can be tricky so it is important to clearly understand what is going
on in these examples.

A key point is that the >> operator will read what it can and then
leave the rest for the next operation.  For example consider the
following code:

char ch;
cin >> ch;

With the input 25.  When the first character, 2, is read in, that is a
single character so it is stored in the ch variable.  But the "5" is
left over for the next operation.  

Look closely at all of the statements in example 3-1 and make sure you
understand the results.

The key point here is that in order to use a predefined function you

The get(), ignore(), putback() and peek() Functions
---------------------------------------------------

These functions are useful for some input tasks but usually the >>
operator will be sufficient.  Of these four functions the get() is the
most useful.  It allows you to read in whitespace.  Remember that the
>> operator skips over whitespace.

The dot Operator
----------------

The book shows the use of the dot operator for the get() (and other)
functions.  It is important to have the dot there.  This pattern will
appear in lots of situations.  If you find yourself getting strange
errors about "cinget()" not found, then check to make sure that you
have "cin.get()".

Input Failure
-------------

Input failures happen when the extraction operator (>>) is unable to
read valid data.  For example:

int a;
cin >> a;

with an input of "W".  In this case since "a" is an integer, the
extraction operator can't put a "W" in a.  It needs a number. Notice
the book example with 67.93.

When there is an input failure the program doesn't crash.  It just
continues without changing the data in the variables.  This is usually
bad. One way to help deal with this is to clear the input.  Notice
example 3-7 and how the clear() function is used to get input into a
good state so that the program can continue.

Output and Formatting Output
----------------------------

Generating output that looks good is a primary focus when writing a
program.  This section discusses several "manipulators" that will help
format output correctly.

Notice that you have to include iomanip in order to use these I/O
manipulators.

setprecision
------------

This manipulator is used to indicate the number of digits to print
after a decimal point.  

fixed
-----

Use this to switch from scientific to fixed point display.  Use the
scientific manipulator to switch back.

showpoint
---------

This manipulator tells the output operator to include the decimal
point even if the part after the point is 0.

setw
----

This is one of the most useful manipulators.  It is used to set the
width of the output of the next expression.

setfill
-------

Use this to fill the blanks from setw() with something other than
space characters.

left and right
--------------

The left and right manipulators change the alignment of the ouput.

Input/Output and the string Type
--------------------------------

You can use the >> operator with strings but remember that the >>
operator skips whitespace.  If you want to read a string value that
has spaces in it you need to use getline().  The syntax for getline()
doesn't use the dot notation.  See the examples in the book.
